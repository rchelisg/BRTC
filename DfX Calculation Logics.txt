DfX Calculation Logics


当 L, N, C 有输入时，在调试窗口标题位置显示 LNC-T，后续运算采用 1LNC-T 路径
当 L, T, N 有输入时，在调试窗口标题位置显示 LTN-C，后续运算采用 2LTN-C 路径
当 L, C, T 有输入时，在调试窗口标题位置显示 LCT-N，后续运算采用 3LCT-N 路径
当 C, T, N 有输入时，在调试窗口标题位置显示 CTN-L，后续运算采用 4CTN-L 路径

路径一旦建立，无论子屏幕上数据怎么改变，都必须维持这个路径，直到数据清零
数据清零后要重新刷新Tcorrected，Ncorrected以及Tcorrected单位的显示，即
  当Tcorrected，Ncorrected为0或者Tcorrected=T，Ncorrected=N时，既不显示数据也不显示单位

长按DfX按钮将数据清零，同时恢复各标签及数据原先的颜色和粗细

计算需要用到以下变量:
L, N, C, T - 这些来自于用户输入，同时也可能成为计算输出值
Voltage，CellType - 对应于两个循环按钮
I， CL - 中间变量
E, Ename, F, Tcorrected, Ncorrected - 计算输出值


添加一个通用计算函数，仅此子屏幕可见
fComp(CellType, L, Voltage, C)
{
  F = 1;			// default. no compensation
  I = L/Voltage;                // working current
  CL = I/(C/1000);              // I in terms of C

  if( CellType==0 )             // tabbed 18650
  {
    F = (0.2/CL)^0.04;
  }
  else if( CellType==1 )        // tabless 18650
  {
    F = (0.2/CL)^0.015;
  }
  else if( CellType==2 )        // no compensation needed
  {
    F = 1;                      // default. no compensation
  }
}

4CTN-L 路径算法
1. Ename = (C/1000)*N*Voltage                      // named energy based on C, N and voltage
2. L = Ename/T                                     // calculate the named load (note, this is before E compensation)
3. Run fComp(CellType, L, Voltage, C) to get F    // get the compensation F
4. E = Ename*F                                    // calculate the actual energy (always smaller than Ename)
5. Lcorrected = E/T                               // derive the actual load allowed
